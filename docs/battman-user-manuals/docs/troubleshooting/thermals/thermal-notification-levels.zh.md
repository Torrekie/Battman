# 温控通知等级

!!! note "已弃用功能"
    此功能已被 iOS 弃用，在 iOS 16 或更高版本中不再可用。

Battman 会展示系统的**温控通知等级**，用于描述 iOS 在发热时会采取多激进的限制措施。

## 等级

热通知系统使用一系列渐进的行为等级，每个等级代表越来越激进的热缓解措施。下方显示的数字是**行为索引**（0-10），而非实际的系统通知等级 ID。

0. **正常** — 正常工作状态，无热限制。

1. **闪光灯衰减至70%** — 手电筒（闪光灯）亮度降至 70%。

2. **背光衰减至70%** — 显示屏背光降至 70%。

3. **闪光灯衰减至50%** — 手电筒亮度进一步降至 50%。

4. **背光衰减至50%** — 显示屏背光降至 50%。

5. **闪光灯已禁用** — 手电筒完全禁用。

6. **背光衰减至25%** — 激进的背光降低至 25%。

7. **地图指南针已禁用** — 禁用地图指南针等视觉效果。

8. **App 终止运行** — 可能会终止应用程序以降低热负载。

9. **设备重新启动** — 极端热条件可能触发设备重启。

10. **已准备好** — 热表已准备好（内部状态）。

## 实现细节

**重要：** 实际的系统通知等级 ID 由 iOS**动态分配**，并非固定值。上方显示的行为索引（0-10）只是不同热缓解行为的标识符。

这种动态映射使 Battman 即使在 iOS 为相同的热行为分配不同数字 ID 时也能正常工作。

## 在 Battman 中

- **温控调校** 页面会展示当前的通知等级，并允许有限的覆盖。
- **硬件温度** 页面可帮助你将这些等级与具体传感器读数对应起来。

如果设备经常处于较高的通知等级，建议降低负载、改善散热，或在温控调校中采用更保守的设置。
